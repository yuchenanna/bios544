1 1. What is Quarto and how do we use it? (2–15 min)

1.1 1.1 What Quarto is

Quarto (.qmd) is a document format that mixes:

  • regular text (what your reader sees),
  • code chunks (R code you run),
  • output (tables/plots/results).

This is how you make reproducible homework, labs, reports, and manuscripts.

1.2 1.2 Key parts of a .qmd

  1. YAML header (the block at the very top between --- lines)
    Controls title, author, output format, etc.
  2. Markdown text
    Headings like ##, bullet lists, bold/italics.
  3. Code chunks
    R code inside fenced blocks like:
# R code goes here

1.3 1.3 Rendering

Rendering means “build” the document into HTML/PDF/Word.

  • In RStudio: click Render (or Knit)
  • If asked to install something, do it once and move on.
Tip

Quarto workflow
1) Write text
2) Add code chunks
3) Render
4) Fix errors (read the first error line carefully)
5) Render again

3 3. Pipes (|>) and the “dplyr sentence” (20–28 min)

3.1 Teaching

A common pattern in R is:

take a dataset → do something → do something → get a result

The pipe lets you write that as a readable chain.

We will use the base R pipe: |>

3.2 Example

penguins |>
  select(species, island, bill_length_mm) |>
  filter(!is.na(bill_length_mm)) |>
  arrange(desc(bill_length_mm)) |>
  head(5)
Tip

Read pipelines out loud:
“Start with penguins, and then select…, and then filter…, and then arrange…”

4 4. filter() operators (28–43 min)

4.1 Teaching: common operators

You’ll use these constantly:

  • Equality: ==
  • Not equal: !=
  • Greater/less: >, <, >=, <=
  • AND / OR: &, |
  • In a set: %in%
  • Missing values: is.na(x) and !is.na(x)
Warning

Common bug: = vs ==
Use == inside filter().

4.2 Operator quick definitions + examples

4.2.1 == (equal)

True when two values match.

penguins |> filter(species == "Adelie")

4.2.2 != (not equal)

True when values do not match.

penguins |> filter(island != "Torgersen")

4.2.3 > (greater than)

Keeps rows with values larger than the threshold.

penguins |> filter(bill_length_mm > 45)

4.2.4 < (less than)

Keeps rows with values smaller than the threshold.

penguins |> filter(body_mass_g < 3500)

4.2.5 >= (greater than or equal)

Includes values equal to the threshold.

penguins |> filter(flipper_length_mm >= 200)

4.2.6 <= (less than or equal)

penguins |> filter(flipper_length_mm <= 180)

4.2.7 & (AND)

Both conditions must be true.

penguins |> filter(island == "Biscoe" & flipper_length_mm >= 200)

4.2.8 | (OR)

At least one condition must be true.

penguins |> filter(species == "Adelie" | island == "Dream")

4.2.9 %in% (one of these values)

Matches any value in a vector.

penguins |> filter(species %in% c("Adelie", "Gentoo"))

4.2.10 is.na(x) and !is.na(x) (missing / not missing)

penguins |> filter(!is.na(body_mass_g))

5 5. mutate() and creating variables (43–55 min)

5.1 Teaching

mutate() creates (or replaces) columns.
Example: converting grams to kilograms.

penguins |>
  mutate(body_mass_kg = body_mass_g / 1000) |>
  select(species, body_mass_g, body_mass_kg) |>
  head(6)

5.2 Task 5.1 — Create two new variables

Create: - body_mass_kg = body_mass_g / 1000 - bill_ratio = bill_length_mm / bill_depth_mm

Then keep only species, sex, body_mass_kg, bill_ratio and show 10 rows.

# TODO: Task 5.1
library(dplyr)
library(palmerpenguins)

penguins %>%
  filter(island != "Torgersen") %>%
  mutate(
    body_mass_kg = body_mass_g / 1000,
    bill_ratio   = bill_length_mm / bill_depth_mm
  ) %>%
  select(species, sex, body_mass_kg, bill_ratio) %>%
  slice_head(n = 10)

If you see Inf or lots of NA, you likely have missing values. Consider filtering missing values before computing ratios.

6 6. group_by() + summarize() (55–72 min)

6.1 Teaching

  • group_by() sets “by-group” behavior
  • summarize() reduces each group to one row

Common summary helpers: - n() = number of rows - mean(x, na.rm = TRUE) - sd(x, na.rm = TRUE) - median(x, na.rm = TRUE)

6.2 Task 6.1 — Average body mass by species

Compute, by species: - n - mean body mass (mean_body_mass_g) - sd body mass (sd_body_mass_g)

Sort from highest mean body mass to lowest.

# TODO: Task 6.1
library(dplyr)
library(palmerpenguins)

penguins %>%
  group_by(species) %>%
  summarize(
    n = n(),
    mean_body_mass_g = mean(body_mass_g, na.rm = TRUE),
    sd_body_mass_g   = sd(body_mass_g, na.rm = TRUE)
  ) %>%
  arrange(desc(mean_body_mass_g))

6.3 Task 6.2 — Two-way grouping

Compute mean flipper length by species and sex, plus n.
Remove missing sex and missing flipper_length_mm first.

# TODO: Task 6.2
library(dplyr)
library(palmerpenguins)

penguins %>%
  filter(!is.na(sex),
         !is.na(flipper_length_mm)) %>%
  group_by(species, sex) %>%
  summarize(
    n = n(),
    mean_flipper_length_mm = mean(flipper_length_mm)
  ) %>%
  arrange(species, sex)

7.1 Task 7.1 — A full pipeline

Create a table that answers:

Among penguins with known sex and body mass, what is the mean body mass (g) by species for penguins with bill length >= 40, and which species has the highest mean?

Requirements: - use |> pipe - use select() at least once - use filter() with >= - use mutate() at least once (any reasonable new variable is fine) - use group_by() + summarize() - use arrange() to sort results

# TODO: Task 7.1
library(dplyr)
library(palmerpenguins)

penguins |>
  filter(flipper_length_mm >= 190) |>                 
  select(species, sex, flipper_length_mm, body_mass_g) |>  
  mutate(                                            
    body_mass_kg = body_mass_g / 1000
  ) |>
  group_by(species, sex) |>                           
  summarize(
    n = n(),
    mean_flipper_length = mean(flipper_length_mm, na.rm = TRUE),
    mean_body_mass_kg   = mean(body_mass_kg, na.rm = TRUE),
    .groups = "drop"
  ) |>
  arrange(desc(mean_body_mass_kg))

A good pattern is:
filter missing → filter bill length → mutate something → group_by(species) → summarize → arrange
